## тЬж Features

<table>
<tr>
<td width="50%" valign="top">

### ЁЯза &nbsp; Gemini Vision Analysis
Upload a photo тАФ AI instantly identifies issue type, severity score (1тАУ10), estimated repair cost in тВ╣, materials needed, and a **butterfly-effect prediction** of what happens if ignored. Zero manual forms.

---

### ЁЯСе &nbsp; Community Verification
Neighbours confirm issues with one tap. **5+ confirmations** auto-escalate to `Verified` status, pushing it to the top of the queue and triggering the dispatch agent.

---

### ЁЯУо &nbsp; Auto Complaint Letters
One tap. Gemini drafts a formal complaint letter addressed to the correct municipal department тАФ pre-filled subject, body, and authority email ready to send.

</td>
<td width="50%" valign="top">

### ЁЯУК &nbsp; City Pulse Insights
AI analyses all reported issues to surface patterns, predict urban hotspots, detect deteriorating zones, and recommend preventive action **before** problems worsen.

---

### тЬЕ &nbsp; AI Resolution Verification
Upload a "fixed" photo тАФ Gemini **visually verifies** the repair matches the original issue before closing it. Prevents fraudulent resolution claims.

---

### ЁЯЧ║я╕П &nbsp; Interactive Map View
Google Maps integration with status-colored SVG markers, dark theme styling, and a spatial overview of every reported issue across the city.

</td>

## тЬж Quick Start

> [!TIP]
> The app works **immediately** with localStorage fallback even without Supabase configured. Gemini API uses graceful mock-data fallback when rate-limited.

### 1. Clone & Install

```bash
git clone https://github.com/itslovepreet1212-stack/civiq.git
cd civiq
npm install
```

### 2. Configure Environment

Create a `.env` file in the project root:

```env
# Supabase
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key

# Google Gemini
VITE_GEMINI_API_KEY=your_gemini_api_key

# Google Maps
VITE_GOOGLE_MAPS_KEY=your_google_maps_api_key
```

### 3. Run

```bash
npm run dev        # тЖТ http://localhost:5173
```

### 4. Build & Test

```bash
npm run build      # Production bundle
npm run preview    # Preview production build
npm test           # Run test suite
```
